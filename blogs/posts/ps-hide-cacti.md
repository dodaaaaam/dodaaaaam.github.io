# 1차 시도

문제의 목표는 다음과 같다.
> 비를 맞지 않거나, 가장 늦게 맞는 구역의 좌측 상단 좌표를 구하라

그래서 나는 이렇게 생각했다.<br>_“각 구역의 **좌측 상단 칸만 관리하면 되지 않을까?**"_


## 🔵 check 배열 생성

가능한 모든 구역의 좌측 상단 셀을 저장하는 2차원 배열을 만든다.

```Python
check_row_num = m-h+1
check_col_num = n-w+1

check = [[0] * (check_col_num) for _ in range(check_row_num)]
```
- `0` → 아직 비를 맞지 않음
- `> 0` → 해당 시점에 비를 맞음

## 🔵 latest 좌표

모든 칸이 비를 맞는 경우를 대비해, 가장 늦게 맞는 좌표를 저장한다.

```Python
latest = [-1, -1]
```

## 🔵 drops 처리

각 drop이 떨어질 때마다 영향을 받는 영역을 계산해서 check 배열을 채운다.

```Python
for j in range(row_min_idx, row_max_num):
    for k in range(col_min_idx, col_max_num):
        if check[j][k] > 0:
            continue
        check[j][k] = i + 1
```
👉 핵심: 해당 drop이 포함되는 모든 구역을 업데이트

## 🔵 결과 찾기

```Python
for i in range(check_row_num):
    for j in range(check_col_num):
        if check[i][j] == 0:
            return [i, j]

return latest
```

# 2차 시도 

시간복잡도를 줄이기 위해 Union-Find (Next Pointer) 방식을 사용한다. 

> 모든 칸을 하나씩 검사하는 것이 아니라, 
> 이미 채운 구간은 점프한다. 
> 💡 **_즉, 각 칸이 "다음 어디로 가야 하는지"를 기억한다. _**

- 기존 방식
: 0 → 1 → 2 → 3 → 4 → 5 → 6

- Union-Find
: 0 → 1 → (2~5 건너뜀) → 6


## 🟢 parent 배열 

parent 배열을 이용하여 check 배열의 각 셀 다음에 와야 할 셀을 지정한다. 

처음에 parent 배열의 각 셀은 자기 자신을 가리키도록 초기화하고,
check 배열의 셀이 채워지면 해당 parent 배열은 다음 셀을 가리키도록 연결한다. 

```Python
parent[r][c] = 다음에 확인할 열
```

## 🟢 칸 채우기 

이미 채워진 check 배열의 셀에 대하여 다음 칸으로 연결한다. 
이때 다음 칸은 find()를 이용하여 탐색한다. 

```Python
def occupy(r, c):
    parent[r][c] = find(r, c + 1)
```

## 🟢 다음 칸 찾기 

이미 채운 칸은 자동으로 건너뛰고 아직 채워지지 않은 가장 가까운 칸을 탐색한다. 
```Python
def find(r, x):
    if parent[r][x] != x:
        parent[r][x] = find(r, parent[r][x])
    return parent[r][x]
```

## ❌ 문제 발생 (시간 초과)

이렇게 Union-Find로 이미 채운 칸을 건너뛰도록 최적화했음에도,
여전히 뒤쪽 테스트케이스에서 시간 초과가 발생했다.

즉, 단순히 “이미 본 칸을 다시 보지 않는 방법”만으로는 부족했고,
애초에 문제를 바라보는 방식 자체를 바꿔야 했다.

### 현재 접근의 한계

지금까지의 접근은 모두 공통적으로 다음 흐름을 따랐다.

drops를 하나씩 보면서 해당 drop이 영향을 주는 모든 구역을 찾아 check 배열을 갱신
즉, 문제를 끝까지 “빗방울 중심” 으로 바라보고 있었던 것이다.

하지만 생각해보니, 이 문제에서 정말 알고 싶은 것은
각 빗방울이 어디에 영향을 주는지가 아니라

> 어떤 선인장 구역이 몇 번째 빗방울에 처음 젖는가

였다.

## 💡 핵심 깨달음 

문제를 다시 생각해보면,
선인장 구역이 처음 비를 맞는 시점은 그 구역 안에 포함된 칸들 중

**_"가장 먼저 비를 맞는 칸의 시점"_** 과 같다.

즉, 어떤 h × w 직사각형 구역이 처음 젖는 시점은
그 구역 내부의 값들 중 최솟값(min) 이다.

그러므로 이 문제는 다음과 같이 바꿔 생각할 수 있다.

> 모든 h × w 직사각형 구역에 대해 **구역 내부의 최소값**을 구하고,
> 그 **최소값이 가장 큰 위치**를 찾는다.
>
> 즉, 이제 이 문제는 더 이상 “선인장 배치 문제”가 아니라
> **2차원 구간 최소값 문제**가 된다.

# 3차 시도 

선인장 구역이 처음 비를 맞는 시점은 해당 구역 내 각각의 칸이 비를 맞는 시점들의 최소값이라는 점을 이용하여 문제를 변환한다. 

모든 h * w 구간의 최소값을 구하기 위해, 
가로 -> 세로 슬라이딩 윈도우를 적용하여 배열을 축소한 뒤, 
최소값이 가장 큰 위치를 선택하여 정답을 구한다. 

## 🟡 시간 배열 만들기

먼저 drops 배열을 이용해서 각 칸이 몇 번째 비에 맞는지를 저장하는 배열을 만든다.

비를 끝까지 맞지 않는 칸은 큰 값(INF)으로 둔다.

```Python
INF = len(drops) + 1
arr = [[INF] * n for _ in range(m)]

for i, (r, c) in enumerate(drops, start=1):
    arr[r][c] = i
```

모든 h * w 구역에 대해 최소값을 구한다면, 시간복잡도가 매우 커져 또다시 시간 초과가 발생한다. 
따라서 이번에는 모든 직사각형을 하나씩 직접 보지 않고, 최솟값을 빠르게 구할 수 있는 방식, 
**_"슬라이딩 윈도우 최솟값"_** 을 적용한다. 

슬라이딩 윈도우의 핵심은 다음과 같다. 

> 한 번 계산한 최소값 정보를 버리지 않고 재사용하자
>
> 직사각형 전체를 한 번에 처리하려고 하면 어렵지만,
> 이를 가로 → 세로 두 단계로 나누면 훨씬 쉽게 구할 수 있다.


## 🟡 1단계: 가로 슬라이딩 윈도우

각 행마다 길이 w 구간의 최소값을 구한 뒤, 중간 배열을 생성한다. 

➡️  결과: m × (n-w+1) 크기의 중간 배열 생성 

이 배열에서 각 칸은 
“그 위치에서 시작하는 가로 길이 w 구간의 최소값”을 의미한다.

```Python
for r in range(m):
    ...
    row_min[c - w + 1] = dq[0][0]
```


## 🟡 2단계: 세로 슬라이딩 윈도우

이제 중간 결과 배열에 대해, 각 열마다 높이 h짜리 구간의 최소값을 구한다.

➡️  결과: (m-h+1) x (n-w+1) 크기의 최종 배열 생성

```Python
for c in range(n - w + 1):
    ...
    final[r - h + 1][c] = dq[0][0]
```

이 배열의 각 값은
해당 위치를 좌측 상단으로 하는 h × w 직사각형 구역의 최소값이 된다.

## 🟡 최종 결과 찾기

최종 배열에서 값이 가장 큰 위치를 선택한다.

값이 같으면 위쪽 → 왼쪽 우선

```Python
for r in range(m-h+1):
    for c in range(n-w+1):
        if final[r][c] > best:
            ...
```
