## 프로젝트 소개

Finders는 분실물과 습득물을 연결해주는 위치 기반 매칭 플랫폼입니다. 사용자가 분실물을 등록하면, 주변에서 등록된 습득물과 자동으로 매칭하여 알림을 보내주는 서비스입니다.

## 주요 기능

- **분실물/습득물 등록**: 사진, 설명, 위치 정보를 포함한 게시글 작성
- **위치 기반 검색**: 카카오맵 API를 활용한 반경 검색
- **자동 매칭 알림**: 키워드 + 위치 기반 매칭 시 푸시 알림
- **실시간 채팅**: 매칭된 사용자 간 1:1 채팅

## 기술 스택

| 영역 | 기술 |
|------|------|
| Frontend | React, React Query, Zustand |
| Backend | Spring Boot, JPA, MySQL |
| Infra | AWS EC2, GitHub Actions |
| API | 카카오맵 API |

## 내가 맡은 역할

프론트엔드 전체를 담당했습니다.

- React 기반 SPA 구조 설계 및 라우팅 구성
- React Query를 활용한 서버 상태 관리 및 캐싱 전략 수립
- 카카오맵 API 연동 및 위치 기반 검색 UI 구현
- Zustand를 사용한 전역 상태(인증, 알림) 관리

## 배운 점

- React Query의 `staleTime`과 `cacheTime`을 적절히 설정하여 불필요한 API 호출을 줄이는 방법을 익혔습니다.
- 지도 API와 React의 렌더링 사이클을 동기화하는 과정에서 `useEffect`의 의존성 관리에 대해 깊이 이해하게 되었습니다.
- 백엔드 팀과 API 명세를 함께 설계하면서 협업 커뮤니케이션의 중요성을 체감했습니다.
