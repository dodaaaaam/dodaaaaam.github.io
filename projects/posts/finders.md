> **Finders**는 필름 카메라 사용자가 **현상소를 탐색하고, 예약하고, 결제까지** 한 번에 처리할 수 있는 서비스입니다. AI 사진 복원, 커뮤니티 피드, 필름카메라 가이드 등 필름 생활 전반을 아우르는 기능을 제공합니다.
>
> 기존 필름 현상 과정에서 정보가 여러 플랫폼에 분산되어 있고, 사진 선택과 인화 요청이 메신저·파일 공유 등 비효율적인 방식으로 이루어지는 문제를 해결하고자 기획되었습니다.

# 🟦 내가 구현한 기능

> 본 프로젝트에서 **사진수다 커뮤니티와 현상관리 기능의 UI 및 사용자 흐름 구현**을 담당하였습니다. 사용자의 행동 흐름에 맞는 인터페이스 설계와 상태 기반 UI 처리를 중심으로 구현하였습니다.

## 🔵 사진수다

#### 1️⃣ 메인 피드 및 검색

- 커뮤니티 피드의 시각적 매력을 높이고 사진이 돋보이도록 **Masonry 기반 비정형 그리드** 레이아웃 구현
- 게시글 검색 기능 구현
  - 제목 / 제목+본문 / 현상소 이름 / 현상소 리뷰 기준 검색
  - 입력값 기반 실시간 필터링
  - 최근 검색어 저장 및 삭제 (개별 / 전체)
  - 입력값 기반 실시간 연관 검색어 표시

#### 2️⃣ 게시글 작성

- 게시글 작성 흐름 설계 및 구현
  - 사진 업로드 → 제목 및 설명 입력 → 현상소 검색 및 선택 → 현상소 리뷰 작성
- OS 기본 파일 선택기를 통해 디바이스 내 이미지 선택 및 업로드 기능 구현
- 현상소 검색 기능 구현
  - 입력값 기반 실시간 필터링
  - 거리순 정렬 및 사용자 위치 기반 거리 표시

#### 3️⃣ 게시글 상세 및 댓글

- 게시글 상세 화면 구성
  - 작성자 정보, 이미지, 내용, 현상소 리뷰 표시
- 사용자 인터랙션 기능 구현
  - 좋아요, 댓글, 공유 기능
- 댓글 UI를 바텀시트 형태로 구현
  - 화면 맥락을 유지하면서 댓글 확인 및 작성 가능
- 사용자 권한 기반 기능 처리
  - 게시글 삭제 (작성자만)
  - 댓글 삭제 및 신고 기능

## 🔵 현상관리

> 필름 현상 과정을 4단계로 구조화하여 **상태 기반 UI와 API 연동을 통해 전체 흐름을 관리하는 기능**을 구현했습니다.

#### 1️⃣ 단계별 상태 UI 및 API 연동

- 서버에서 전달받은 현상 상태 데이터를 기반으로 각 단계별 UI를 동적으로 렌더링
- 진행 상태 및 완료 시간 등 **API 응답 데이터를 UI와 매핑하여 사용자에게 직관적으로 전달**

#### 2️⃣ 사용자 선택 기반 분기 처리

- 인화 여부 선택에 따라 API 요청 및 이후 흐름을 분기 처리
  - 인화 진행
  - 프로세스 종료
- 배송 / 직접 수령 선택에 따라 **서버 데이터와 연동된 조건 분기 UI 구현**

#### 3️⃣ 사용자 액션 및 API 처리

- 사진 다운로드 기능 구현 (서버 데이터 기반)
- 수령 확정, 배송 조회 등 **사용자 액션에 따른 API 요청 및 상태 업데이트 처리**
- "사진 자랑하러 가기" 기능을 통해 현상관리 → 사진수다로 사용자 흐름 연결

#### 4️⃣ 상태 기반 인터페이스 설계

- 현상 단계와 사용자 선택값, API 응답 상태를 조합하여 화면이 동적으로 변경되는 구조 설계
- 하나의 흐름 안에서 다양한 시나리오를 처리할 수 있도록 **UI와 데이터 상태를 일관되게 관리**
